# Cloud Ops Hub Web PDR

## 1. 文档目标

本文件用于替换旧版前端看板说明，重新定义 Cloud Ops Hub 门户首页的结构、视觉边界与模块职责。新版方案保留参考图中的蓝白云岛风格与 Bento Grid 气质，但整体明显精简，首页只承担“个人名片 + 四个模块入口 + 一句个人谏言”的职责。

目标是做到：

- 首屏进入即看全，不依赖滚动理解信息结构
- 模块边界清晰，不把“应用”和“运维”混在一起
- 首页更像个人作品门户，而不是复杂运维大盘
- 页面整体缩放更克制，避免首屏过满
- 后续可逐步接真实接口，但首页结构不再频繁变动

## 2. 产品定位

这是 Deriou 的个人云原生门户首页，用于同时承载：

- 个人身份展示
- 博客内容入口
- GitHub 作品集入口
- 应用集合入口
- 运维与管理入口

首页不是业务工作台，不承载复杂筛选、长列表、日志流、图表联动等高密度操作。它首先是一个简洁、可信、带个人气质的“总入口”。

## 3. 首页信息架构

首页固定为以下区域，且在桌面端应尽量控制在一屏内完成展示：

1. 顶栏导航
2. 左侧个人信息卡
3. 博客模块卡
4. 项目展示模块卡
5. 应用集合模块卡
6. 运维模块卡
7. 底部个人谏言卡

说明：

- 个人信息卡占左侧主列
- 右侧为四个功能模块的 2x2 卡片布局
- 谏言卡横向铺在四个模块下方
- 顶栏简洁，不放搜索框、不放二级菜单、不放复杂状态区
- 谏言区域使用正常大标题字号即可，不再使用过大的展示字

## 4. 模块职责定义

### 4.1 博客

职责：

- 展示博客模块入口
- 显示文章总数
- 展示最新一篇文章摘要
- 点击后进入博客内容页

首页卡片内容建议：

- 模块标题：博客
- 数量标签：如 `87 Posts`
- 最新文章标题
- 更新时间
- 一个简洁跳转动作

首页不放：

- 标签云
- 分类列表
- 多篇文章列表
- 搜索框

补充说明：

- 首页暂时只放 Blog 入口
- 博客依旧属于 Gateway-Portal 应用管理托管范围，但首页仍保留单独的博客展示卡

### 4.2 项目展示

职责：

- 作为 GitHub 作品集展示入口
- 以仓库卡片形式展示当前代表项目
- 点击后直接跳转到 GitHub 仓库

当前确认内容：

- 目前只展示一个仓库：`Deriou/Cloud-ops-hub`

首页卡片内容建议：

- 模块标题：项目展示
- 一个代表仓库卡片
- 展示仓库名、简短说明、GitHub 外链标识
- 卡片底部提供 `View Repo` 或 `View on GitHub` 入口

首页不放：

- 提交记录图
- Star/Fork 实时统计
- 技术筛选器

### 4.3 应用集合

职责：

- 只放各类“应用或服务入口”
- 偏使用向，不带管理属性

当前确认内容：

- 首页暂时只放一个应用入口：`Portal`
- `Portal` 的职责文案为：
  - 有哪些应用
  - 状态如何
  - 从哪进

明确不包含：

- Blog 入口
- Grafana
- Jenkins
- 带管理权限和鉴权要求的后台入口

首页卡片内容建议：

- 模块标题：应用集合
- 当前仅一个入口卡片
- 每个入口只展示名称与一句说明

### 4.4 运维

职责：

- 承载管理向与观测向入口
- 聚合 Gateway / Blog 的管理能力与健康状态
- 为后续鉴权设计预留边界

明确包含：

- Gateway 管理入口
- Blog 管理入口
- 健康状态摘要
- Jenkins 入口
- Grafana 入口

首页卡片内容建议：

- 模块标题：运维
- 一个简洁状态标签，如 `Stable`
- 当前整体状态一句话摘要
- 2 到 4 个管理/观测入口
- 说明该区域后续会接入鉴权

首页不放：

- 实时日志流
- 大表格
- 复杂监控图表

## 6. 视觉与交互规范

### 6.1 保留的风格

- 蓝白主色调
- 浅背景
- 圆角云岛卡片
- 轻阴影
- Bento Grid 排布
- 简洁科技感

### 6.2 必须精简的部分

- 不做重毛玻璃
- 不做粒子背景
- 不做 3D 云朵
- 不做复杂动态边框
- 不做过多状态灯和装饰图层
- 不做过密的信息堆叠

### 6.3 页面感觉

目标视觉关键词：

- 干净
- 轻
- 稳
- 有个人品牌感
- 第一眼就能看完
- 比当前原型再小一档

交互只保留：

- 卡片 hover 浮起
- 按钮 hover
- 状态点轻微脉冲

## 7. 顶栏规范

顶栏采用非常简洁的个人门户导航形式。

当前确认结构：

- 左侧：仅标题 `Deriou的个人编程笔记`
- 中间/右侧：`Home`、`Blog`、`Projects`、`Apps`、`Ops`
- 最右：主按钮为 `GitHub`

约束：

- 顶栏高度控制紧凑
- 不放搜索框
- 不放用户菜单
- 不放复杂切换器
- 左上角不再使用图标

## 8. 布局与缩放要求

桌面端是本次设计主场景，必须优先保证 1440px 左右宽度下首屏完整展示。

强约束：

- 首屏进入尽量不滚动即可看到全部核心内容
- 整体间距比上一版更紧凑
- 卡片高度要克制，避免模块过高
- 文案长度要短，避免撑开布局
- 谏言区域字号使用常规大标题级别即可，不再使用超大展示字

响应式要求：

- 桌面端使用左 1 右 4 模块 + 底部谏言布局
- 平板端允许分两屏，但首屏仍应看到个人信息与至少两个模块
- 移动端允许纵向堆叠，但保留同样的信息顺序

## 9. 数据映射建议

首页只消费轻量摘要数据，不直接接复杂明细。

当前映射：

- 博客：文章总数 + 最新文章
- 项目展示：`Deriou/Cloud-ops-hub`
- 应用集合：单一 `Portal` 入口
- 运维：健康摘要 + Gateway / Blog 管理入口 + Jenkins / Grafana 预留
- 个人信息：静态配置

首页不要依赖这些复杂数据：

- 全量文章分页
- 全量应用注册表列表
- 全量工作负载清单
- 实时流水线日志

## 10. 后续页面关系

首页只负责“总览和入口”，真正的内容与操作进入各模块详情页：

- 博客卡片进入 Blog 页面
- 项目展示卡片进入 GitHub 仓库
- 应用集合卡片进入 Portal 页面
- 运维卡片进入 Ops 页面

因此首页卡片必须短、准、可点击，不再承担详情页职责。

## 11. 鉴权预留

鉴权只在运维模块预留，不在首页显式展开复杂权限设计。

当前约定：

- 应用集合默认面向访问入口
- 运维模块未来区分 Guest 与 Admin
- Jenkins / Grafana / 管理后台默认属于受控入口

首页表现建议：

- 只显示“管理入口”语义
- 不在首页做登录弹窗和权限流程

## 12. 个人谏言

当前确认文案：

要求：

- 首页底部保留一张横向谏言卡
- 使用正常大标题字号
- 保持蓝底白字，但不再占据过大的视觉体积

## 13. 本版设计结论

新版首页应是一个精简后的个人门户，而不是完整控制台。它保留蓝白 Bento 气质，但内容只围绕五件事组织：

- 我是谁
- 我写什么
- 我做过什么
- 我有哪些应用
- 我如何管理这些系统

首页完成标准：

- 首屏可看全核心信息
- 四个模块边界清楚
- 运维和应用不混放
- 信息量明显低于旧版
- 视觉保留参考图风格，但整体更克制、更实用
